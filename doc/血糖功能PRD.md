# 血糖监测功能 PRD

## 1. 背景与目标

### 背景
若依管理系统增加血糖监测模块，帮助用户记录和追踪血糖数据。

### 目标
- 快速记录每日血糖数据
- 直观展示血糖趋势
- 提醒用户血糖异常

## 2. 用户画像

- **主要用户**：糖尿病患者、健康管理者
- **使用场景**：每日多次测量血糖并记录
- **核心诉求**：简单记录、趋势可视化、异常提醒

## 3. 功能需求

### 3.1 首页改造
将原 Dashboard 首页改造为血糖数据展示页。

**页面元素**：
1. **统计卡片**（顶部3个）
   - 今日平均血糖值
   - 今日最高值
   - 今日最低值
   
2. **今日血糖曲线**
   - 折线图
   - X轴：时间点
   - Y轴：血糖值(mmol/L)
   - 正常范围参考线（3.9~7.8）

3. **近7天趋势**
   - 柱状图
   - 每天平均血糖值

4. **血糖记录列表**
   - 列：时间、血糖值、类型（空腹/餐前/餐后）、备注
   - 支持添加新记录
   - 支持删除

### 3.2 血糖记录
- **字段**：血糖值(mmol/L)、测量时间、类型、备注
- **类型选项**：空腹、餐前、餐后
- **数据范围**：1~35 mmol/L

### 3.3 血糖统计
- **今日统计**：记录数、平均值、最大值、最小值
- **7天趋势**：每天平均血糖值

## 4. 页面交互

| 操作 | 说明 |
|------|------|
| 查看首页 | 自动加载今日统计和曲线 |
| 添加记录 | 点击"添加记录"按钮，弹出表单 |
| 删除记录 | 每行右侧删除按钮，二次确认 |

## 5. API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| /system/bloodSugar/list | GET | 血糖记录列表（分页） |
| /system/bloodSugar/todayStats | GET | 今日统计 |
| /system/bloodSugar/sevenDaysTrend | GET | 近7天趋势 |
| /system/bloodSugar | POST | 新增记录 |
| /system/bloodSugar | PUT | 修改记录 |
| /system/bloodSugar/{id} | DELETE | 删除记录 |

## 6. 数据字典

### 餐前/餐后类型（meal_type）
| 值 | 名称 |
|----|------|
| 0 | 空腹 |
| 1 | 餐前 |
| 2 | 餐后 |

### 血糖参考范围
| 范围 | 说明 |
|------|------|
| < 3.9 mmol/L |偏低（低血糖警告）|
| 3.9 ~ 7.8 mmol/L | 正常 |
| > 7.8 mmol/L | 偏高 |

## 7. 验收标准

- [ ] 首页显示今日统计卡片（3个）
- [ ] 今日血糖曲线折线图正常渲染
- [ ] 7天趋势柱状图正常渲染
- [ ] 血糖记录列表正常显示
- [ ] 可正常添加血糖记录
- [ ] 可正常删除血糖记录
- [ ] API 接口返回数据正确

## 8. 风险与依赖

- **依赖前端**：ECharts 图表库（CDN引入）
- **依赖后端**：血糖模块 API 开发
- **依赖数据库**：sys_blood_sugar 表创建
